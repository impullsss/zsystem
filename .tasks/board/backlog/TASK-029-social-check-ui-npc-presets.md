# TASK-029 — Социальные проверки: UI, пресеты NPC, макрос/ПКМ

**Статус:** backlog  
**Приоритет:** medium  
**Размер:** M  
**Зависит от:** TASK-023, TASK-024  

## Цель

Сделать социальные проверки быстрыми и не-метагеймовыми:
- игрок делает проверку в 1–2 клика
- параметры “настроение/отношение” и базовая сложность задаются ГМом на NPC и/или скрыты от игрока

## UX (рекомендация)

### Режим игрока (быстро)

- Окно “Социальная проверка”
- Минимум полей:
  - навык (убеждение/запугивание/торг и т.п.)
  - цель (выбранный токен NPC, если есть)
- Кнопка “Бросить”
- В чат игроку: успех/провал + короткий текст (“сложно”, “опасно”), без цифр факторов

### Режим ГМа (расширенный)

- ГМ видит/может править факторы сложности
- Может “применить и бросить” или просто “применить” (для последующих бросков)
- В чат ГМу: полный breakdown (chance, roll, факторы)

## Пресеты на листе NPC (для ГМа)

Добавить поля (или флаги) на NPC:
- `attitude`: friendly / neutral / hostile
- `socialPreset`: easy / normal / hard (или числовой множитель)
- (опционально) `dispositionNotes` — заметка только ГМу

Эти значения используются как часть `autoDifficulty` при социальных проверках.

## Быстрый запуск

- Макрос “Social Check”:
  - если выделен токен NPC → берём его пресеты
  - если нет → открываем выбор цели/проверку без пресета
- Вариант: пункт в контекстном меню токена (ПКМ) → “Социальная проверка”

## Требования

- Не требовать от игрока выставлять числа/штрафы вручную
- Не раскрывать настроение/отношение NPC игроку, если настройка мира “скрывать факторы” включена
- Константы факторов вынести в одно место (настройки/константы) для тюнинга

## Проверка

- Игрок: 1–2 клика → результат в чат
- ГМ: может быстро настроить отношение NPC и увидеть breakdown
- При смене NPC (другой токен) подтягиваются его пресеты автоматически
