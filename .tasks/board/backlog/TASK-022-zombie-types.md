# TASK-022 — Типы зомби

**Статус:** backlog
**Приоритет:** low
**Размер:** L

## Цель

Добавить разнообразие зомби с разными поведениями, статами и способностями.

## Идеи типов

| Тип | Особенность |
|-----|-------------|
| **Обычный** | Текущий зомби, ничего нового |
| **Бегун** | AGI 6–8, AP 6, атакует дважды за ход |
| **Громила** | STR 9–10, VIG 8–10, медленный (AP 3), большой урон |
| **Кричун** | Низкий урон, при атаке добавляет +20 шума → агрит других |
| **Раздутый** | При смерти — взрыв (AoE урон 2d6 вокруг) |
| **Заражённый** | Каждый удар — бросок на инфекцию у цели |

## Реализация

- Новое поле `system.zombieType` (string) на листе зомби
- В `_onCreate` при рандомизации — случайный выбор типа или по выбору ГМа
- Лист зомби показывает тип с иконкой
- `onTurnStart` и `applyDamage` проверяют тип для спец-поведения

## Требует уточнения

- Какие типы из `GAME_RULES.md` приоритетны?
- Нужен ли ИИ (автоход зомби) или только статы?
