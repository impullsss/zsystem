# TASK-028 — Прокачка навыков под Fallout-модель (0–200+, без статов в точности)

**Статус:** backlog  
**Приоритет:** high  
**Размер:** M  
**Зависит от:** TASK-023, TASK-024, TASK-025  

## Цель

Подогнать систему прогрессии под модель попадания `chance = clamp(skill - autoDifficulty, 5, 95)`:
- навыки можно качать **выше 100**
- характеристики **не участвуют** в точности
- рост навыка остаётся полезным даже при капе шанса (через качество/оверкап)

## Решение по максимуму навыка (рекомендация)

Сделать **настраиваемый максимум** для мира:
- `zsystem.maxSkill = 200` (дефолт, удобно для UI/баланса)
- при этом код не должен “ломаться” если значение >200 (перки/предметы/книги могут дать временный оверкап)

## Роль характеристик (без влияния на попадание)

- **STR**: урон ближнего боя, требования оружия, переноска
- **AGI**: AP, движение, уклонение
- **PER**: обнаружение/спот/инициатива; (опционально) уменьшение штрафов видимости/дальности, но НЕ прямой бонус к chance
- **INT**: скорость обучения (скилл-поинты/XP), наука/медицина/крафт
- **CHA**: мораль/лидерство/торговля/соц
- **VIG**: выносливость/сопротивления/пороги травм (в связке с TASK-026)

## Скилл-поинты и стоимость (предложение)

Добавить универсальную формулу стоимости повышения навыка (ступеньки по диапазонам):
- 0–100: cost = 1
- 101–150: cost = 2
- 151–200: cost = 3
- 201+: cost = 5 (оверкап, если разрешён)

Альтернатива (если хочется плавнее): `cost = 1 + floor(skill / 50)` (кап по настройке).

## Почему качать >100 имеет смысл

Даже при `maxChance=95` рост навыка:
- помогает пробивать высокую `autoDifficulty` (укрытие/дальность/called shot)
- увеличивает **quality** (через `margin`, криты, антифамбл) — см. TASK-025

## Что изменить в системе

- UI навыков: ввод/отображение >100 и (дефолт) до 200
- Валидация данных: clamp к `maxSkill` (если включено), но модификаторы могут давать временный оверкап
- Экран/логика распределения очков (если есть): учёт стоимости по диапазонам

## Проверка

- Можно прокачать навык с 40 до 120 без побочных эффектов в листе/подготовке данных
- При `maxSkill=200` значение не превышает 200 от “базы”, но баф/перк может временно поднять `effectiveSkill` выше
- Логи/чат показывают эффект повышения навыка (через рост `chance` или `margin/crit`)
