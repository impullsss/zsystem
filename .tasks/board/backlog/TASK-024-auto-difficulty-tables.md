# TASK-024 — Таблицы автосложности (range/cover/called shot/visibility)

**Статус:** backlog  
**Приоритет:** high  
**Размер:** M  
**Зависит от:** TASK-023  

## Цель

Определить математически логичные и игровые приятные таблицы/формулы `autoDifficulty`, чтобы:
- игрок **не промахивался постоянно**
- но и не "попадал всегда", сохраняя риск
- вызов по частям тела (called shot) был осмысленным выбором

## Что входит в autoDifficulty

### Дальность

- 0…`weapon.range` → минимальный штраф
- дальше `weapon.range` → растущий штраф (ступеньками или формулой)

### Укрытие

- нет / половина / полное / вне LOS

### Помехи (токены между)

- штраф за каждый перекрывающий токен (с капом)

### Видимость/свет

- дневной / сумерки / ночь / слепота

### Called shot по части тела

- torso: 0
- arms/legs: средний штраф
- head: большой штраф

### Уклонение цели

- уклонение цели конвертируется в difficulty (а не вычитается из skill отдельно)

## Требования

- Все штрафы должны быть выражены в одной шкале "difficulty" (проценты)
- Должны быть дефолты (константы), пригодные для универсальной системы (не только зомби)
- Возможность настраивать ключевые коэффициенты через `game.settings` (минимум: called shot, cover, дальность)

## Проверка

- В реальных типичных боях (5–10 клеток, половина укрытия, без called shot) шанс попадания у среднего стрелка должен быть в районе 55–75%
- Голова должна быть рискованной (обычно 20–45%) без прицеливания, но достижимой с прицеливанием/перками
